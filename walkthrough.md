# VibeESP32 - 파일 및 폴더 일괄 관리, 실행 엔진 및 테트리스 템플릿 추가 보고서

ESP32 보드의 하위 디렉토리를 포함한 전체 파일 시스템을 재귀적으로 조회하고, 로컬 컴퓨터의 라이브러리 폴더(`lib` 등) 자체를 구조 그대로 통째로 업로드할 수 있는 기능, 강제 인터럽트 헬퍼, 그리고 요청하신 "테트리스 게임" 추천 템플릿 추가 작업을 완료했습니다.

---

## 🛠️ 추가 및 개선 사항 요약

### 1. "🎮 테트리스 게임" 추천 템플릿 단추 추가
- **기능**: 프론트엔드의 `추천 템플릿` 영역에 `🎮 테트리스 게임` 버튼을 새로이 구성했습니다.
- **코드 무결성**:
  - 작성해주신 테트리스 파이썬 코드 전체(하드웨어 I2C/Touch 핀맵 설정, 블록 크기 최적화, 우측 UI 텍스트 오프셋 조정 등)를 **단 하나의 글자나 주석 누락 없이 100% 동일하게** 템플릿 백엔드 데이터에 탑재시켰습니다.
- **연동 및 라우팅**:
  - `suggestions` 배열 및 `api/chat.ts` API 백엔드 핸들러 라우팅을 추가했습니다.
  - "테트리스", "tetris", "게임" 등의 관련 키워드 감지 시, OpenAI API를 호출하지 않고 로컬에 내장된 테트리스 소스코드 및 코드 상세 해설 리스트를 청크 형태로 즉시 사용자에게 스트리밍 노출시킵니다.
  - 이로써 OpenAI 서버 오류나 API Key 유무와 무관하게 테트리스 템플릿의 소스코드를 즉시 생성하고 확인하여 실행할 수 있습니다.

### 2. 코드 실행 안정성 고도화 (Thonny 방식 파일 실행)
- **해결**: 실행할 코드를 보드 내부의 임시 파일인 `_run.py`로 **Base64 청크 방식**으로 안전하게 선(先) 전송하여 온전한 무결성 파일로 저장한 뒤 `exec(open('_run.py').read(), globals())` 1라인 명령어로 실행시킵니다.
- **안정성 강화**: 실행 전 `Ctrl+C`를 보내 프롬프트 라인의 잔여 문자들을 지우고 명령 수신 시 `\r\n` 개행을 전달하여 즉시 인터프리터 작동을 유도합니다.

### 3. 무한 루프 강제 종료 및 Raw REPL 진입 안정화 (`enterRawREPL`)
- **해결**: `enterRawREPL` 공용 헬퍼 함수를 추가하여, 보드 제어 시작 시 연속적으로 `Ctrl+C`를 두 번 이상 강력히 보내 무한 루프를 완전히 종료시킨 뒤 `Ctrl+A`를 전송하도록 단일화했습니다.

### 4. 폴더 업로드 기능 구현 (Bulk Folder Upload)
- **HTML5 Directory API 연동**: 사용자가 폴더를 선택할 수 있도록 `<input type="file" webkitdirectory directory multiple>` 속성을 도입했습니다.

### 5. VFS 재귀적 전체 파일 목록 조회
- **트리 구조 목록 파싱**: 기존 단층 수준의 `os.listdir()` 호출 방식 대신, 하위 디렉토리를 깊이 우선 탐색(DFS)하여 모든 하위 파일을 상대 경로와 함께 수집하는 MicroPython 재귀 함수(`_list_all`)를 정의하여 쿼리하도록 수정했습니다.
